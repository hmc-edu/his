## 1. 你需要先掌握的最少 Vue 3 知识

如果你完全没接触过 Vue，先去官方教程过一遍：<https://cn.vuejs.org/tutorial/>。这里只列项目里实际用到的几个核心概念：

### 1.1 单文件组件（SFC）

每个 `.vue` 文件就是一个组件，由三块组成：

```vue
<template>
  <!-- HTML 模板 -->
  <div>{{ msg }}</div>
  <button @click="onClick">点击</button>
</template>

<script setup>
// JavaScript：响应式数据 + 函数
import { ref } from 'vue'

const msg = ref('Hello')        // ref：把值变成响应式的
function onClick() {
  msg.value = 'World'           // 改 .value 触发模板重新渲染
}
</script>

<style scoped>
/* 仅作用于本组件的 CSS */
button { color: red; }
</style>
```

### 1.2 响应式 API

| API | 用途 | 例子 |
| --- | --- | --- |
| `ref(0)` | 单个值 | `const count = ref(0); count.value++` |
| `reactive({...})` | 对象 | `const form = reactive({name:'', age:0})` |
| `computed(...)` | 派生值 | `const isAdmin = computed(() => userStore.role === 'ADMIN')` |
| `watch(src, cb)` | 监听变化 | `watch(count, (n,o) => console.log(n))` |
| `onMounted(...)` | 生命周期：挂载完成 | `onMounted(() => load())` |

### 1.3 模板里的指令

| 指令 | 作用 | 例子 |
| --- | --- | --- |
| `{{ x }}` | 文本插值 | `<span>{{ user.name }}</span>` |
| `:prop="x"` | 属性绑定 | `<el-input :disabled="loading" />` |
| `@event="fn"` | 事件监听 | `<button @click="onLogin">登录</button>` |
| `v-model="x"` | 双向绑定 | `<el-input v-model="form.username" />` |
| `v-if="cond"` | 条件渲染 | `<div v-if="user">已登录</div>` |
| `v-for="x in list"` | 列表渲染 | `<li v-for="d in drugs" :key="d.id">{{ d.name }}</li>` |

## 3. 一次操作背后跑了哪些代码（举例：登录）

```
用户在 Login.vue 输入 admin/123456，点"登录"
    │
    │  @click="onLogin"
    ▼
Login.vue#onLogin()
    │
    │  await userStore.login({ username, password })
    ▼
stores/user.js#login()
    │
    │  await loginApi(payload)
    ▼
api/auth.js#loginApi()
    │
    │  request.post('/api/auth/login', data)
    ▼
api/request.js (axios 实例)
    │
    │  请求拦截器：注入 Authorization 头
    │
    ▼
浏览器发出 HTTP POST → /api/auth/login
    │
    │  Vite Dev Server 代理 → http://localhost:8080
    ▼
[后端处理...]
    │
    ▼
浏览器收到响应 { code:0, data:{ tokenValue, userInfo } }
    │
    │  响应拦截器：code==0 直接返回 data
    ▼
api/auth.js loginApi 返回 data
    │
    ▼
stores/user.js#login() 把 token 存到 state + localStorage
    │
    ▼
Login.vue#onLogin() 跳转到 /dashboard
```

## 4. 三个核心机制

### 4.1 Pinia store —— 跨页面共享状态

`stores/user.js` 是全局唯一的"用户状态盒子"：

```js
import { defineStore } from 'pinia'

export const useUserStore = defineStore('user', {
  state: () => ({
    token: '',
    user: null
  }),
  getters: {
    isLogin: (s) => !!s.token,
    role: (s) => s.user?.role || ''
  },
  actions: {
    async login(payload) {
      const res = await loginApi(payload)
      this.token = res.tokenValue
      this.user  = res.userInfo
    },
    clear() {
      this.token = ''
      this.user  = null
    }
  }
})
```

任何页面里：

```js
import { useUserStore } from '@/stores/user'
const userStore = useUserStore()

console.log(userStore.role)         // 直接访问 state / getter
userStore.login({...})              // 调 action
userStore.clear()                   // 退出
```

### 4.2 axios 拦截器 —— 自动处理 token 与异常

`api/request.js` 是所有 HTTP 请求的"中间层"：

```js
import axios from 'axios'

const request = axios.create({ baseURL: '/', timeout: 15000 })

// 请求拦截器：每个请求自动带 token
request.interceptors.request.use(config => {
  const userStore = useUserStore()
  if (userStore.token) {
    config.headers['Authorization'] = userStore.token
  }
  return config
})

// 响应拦截器：自动解包 R<T>
request.interceptors.response.use(
  response => {
    const { code, msg, data } = response.data
    if (code === 0) return data           // 成功，业务方拿到的就是 data
    if (code === 401) {
      userStore.clear()
      router.replace('/login')
    }
    ElMessage.error(msg)
    return Promise.reject(new Error(msg))  // 业务方走 catch
  },
  error => {
    ElMessage.error(error.message || '网络异常')
    return Promise.reject(error)
  }
)

export default request
```

所以业务代码这么写：

```js
// 不需要判断 code，不需要 try-catch（除非你要做特殊处理）
const reg = await createRegistration({ patientId, doctorId })
result.value = reg                    // 直接拿 data
```

### 4.3 路由守卫 —— 登录拦截 + 角色过滤

`router/index.js`：

```js
router.beforeEach((to) => {
  const userStore = useUserStore()

  // 1. 公开页面（如 /login）直接放行
  if (to.meta?.public) return true

  // 2. 未登录 → 跳登录页
  if (!userStore.isLogin) return { path: '/login', query: { redirect: to.fullPath } }

  // 3. 角色不匹配 → 跳 dashboard
  if (to.meta?.roles && !to.meta.roles.includes(userStore.role)) {
    return { path: '/dashboard' }
  }

  return true
})
```

每个路由的 meta 里声明该页可以由哪些角色访问：

```js
{
  path: 'system/user',
  component: () => import('@/views/system/User.vue'),
  meta: { title: '用户', roles: ['ADMIN'], group: '系统管理' }
}
```

## 5. 加一个新页面的 6 步法

以"我要加一个 **检查申请** 页面"为例：

### Step 1 — 写 API 文件

`frontend/src/api/examOrder.js`：

```js
import request from './request'

export const listExamOrders   = (params) => request.get('/api/exam-orders', { params })
export const createExamOrder  = (data)   => request.post('/api/exam-orders', data)
export const updateExamOrder  = (id, d)  => request.put(`/api/exam-orders/${id}`, d)
export const deleteExamOrder  = (id)     => request.delete(`/api/exam-orders/${id}`)
```

### Step 2 — 写页面组件

`frontend/src/views/doctor/ExamOrder.vue`（仿照 `Drug.vue`）：

```vue
<template>
  <el-card class="page-card">
    <div class="page-toolbar">
      <el-input v-model="visitId" placeholder="就诊单 ID" style="width: 180px" />
      <el-button type="primary" @click="load">查询</el-button>
      <el-button type="success" @click="openCreate">新增检查申请</el-button>
    </div>

    <el-table :data="records" border stripe>
      <el-table-column prop="id"         label="ID"         width="80" />
      <el-table-column prop="itemName"   label="检查项目"   />
      <el-table-column prop="status"     label="状态"       width="100" />
      <el-table-column prop="orderedAt"  label="开单时间"   width="180" />
    </el-table>

    <el-dialog v-model="dialogVisible" title="新增检查申请">
      <el-form :model="form" label-width="80px">
        <el-form-item label="就诊单 ID">
          <el-input-number v-model="form.visitId" />
        </el-form-item>
        <el-form-item label="检查项目">
          <el-input v-model="form.itemName" placeholder="如：胸部 X 光" />
        </el-form-item>
      </el-form>
      <template #footer>
        <el-button @click="dialogVisible = false">取消</el-button>
        <el-button type="primary" @click="onSave">保存</el-button>
      </template>
    </el-dialog>
  </el-card>
</template>

<script setup>
import { onMounted, reactive, ref } from 'vue'
import { ElMessage } from 'element-plus'
import { listExamOrders, createExamOrder } from '@/api/examOrder'

const records = ref([])
const visitId = ref(null)
const dialogVisible = ref(false)
const form = reactive({ visitId: null, itemName: '' })

async function load() {
  records.value = await listExamOrders({ visitId: visitId.value })
}
function openCreate() {
  Object.assign(form, { visitId: null, itemName: '' })
  dialogVisible.value = true
}
async function onSave() {
  if (!form.visitId || !form.itemName) return ElMessage.warning('请填完整')
  await createExamOrder(form)
  ElMessage.success('已保存')
  dialogVisible.value = false
  load()
}

onMounted(load)
</script>
```

### Step 3 — 注册路由

`frontend/src/router/index.js`，在 children 里加：

```js
{
  path: 'doctor/exam-orders',
  name: 'doctor-exam-orders',
  component: () => import('@/views/doctor/ExamOrder.vue'),
  meta: { title: '检查申请', icon: 'Document', roles: ['ADMIN', 'DOCTOR'], group: '医生工作站' }
}
```

侧栏菜单是根据路由 meta 自动生成的（`DefaultLayout.vue` 里读 `meta.group`/`meta.title`/`meta.icon`），所以不用单独加菜单代码。

### Step 4 — 验证

启动前端 `pnpm dev`，用 doctor 账号登录。侧栏"医生工作站"分组下应该出现"检查申请"菜单。点进去看页面。

### Step 5 — 调接口确认

F12 打开 Network 面板，刷新页面：

- 应看到一条 `GET /api/exam-orders?visitId=` 请求
- 状态码 200，返回 `{code:0, msg:"ok", data:[...]}`

### Step 6 — 调样式

如果页面看起来歪歪扭扭：

- 检查 `<el-card class="page-card">` 是否套了
- 检查 `<div class="page-toolbar">` 是否包住按钮组
- 这些样式定义在 `src/styles/index.css`

## 7. 调试技巧

### 7.1 浏览器 DevTools

按 F12，几个常用面板：

- **Console**：看 `console.log` 输出 / 报错堆栈
- **Network**：看每个 HTTP 请求的 URL / 状态码 / 请求头 / 响应体
- **Elements**：右键页面元素 → "检查"，看 DOM 树和样式
- **Application → Local Storage**：看 `his_auth` 里存的 token

### 7.2 Vue Devtools

Chrome 扩展：搜索 "Vue.js devtools"，安装后 F12 多一个 "Vue" 面板：

- 看每个组件的 props / state / computed
- 看 Pinia store 的实时数据
- 看 Vue Router 的当前路由

### 7.3 console.log 是你最好的朋友

```js
async function load() {
  const res = await pagePatients({ keyword: keyword.value })
  console.log('拉到的数据：', res)        // 这句让你看到底拉到了什么
  records.value = res.records
}
```

### 7.4 mock 后端响应

后端没准备好时，可以临时在 api 文件里返回假数据：

```js
export const listExamOrders = async () => {
  // return await request.get(...)
  return [{ id: 1, itemName: '胸部 X 光', status: 'PENDING' }]
}
```

## 8. 常见前端"坑"清单

| 坑 | 现象 | 解决 |
| --- | --- | --- |
| 改了 ref 但不更新 | 直接赋值数组/对象 | 用 `arr.value = [...]` 或修改 `.value` |
| `Cannot read property of undefined` | 异步还没拿到数据时模板就用了 | 模板里加 `v-if="user"` 或用 `?.` 链式 |
| F5 刷新后丢失登录 | localStorage 没存 | 检查 `userStore.persist()` 是否调用 |
| 跨域报错 CORS | 没走代理 / 后端 CORS 没开 | 确认 `vite.config.js` 代理配置 |
| 表单 v-model 不生效 | 用了 `:value=` 而不是 `v-model` | 改用 `v-model` |
| 按钮多次点击 | 没加 loading 锁 | `loading.value = true; try {...} finally { loading.value = false }` |
| `process is not defined` | 用了 Node.js API | 浏览器端不能用 `process.env.XXX`，改用 `import.meta.env.XXX` |

## 9. 编码规范

- 缩进 2 空格
- `import` 顺序：vue → 第三方 → `@/` 项目内 → 相对路径
- 单引号 `'` 不用双引号 `"`
- 函数式 + 组合式 API（`<script setup>`），不用 Options API
- 空行：`import` 后空一行，函数之间空一行
- 不写 `interface` / TypeScript 类型注解（项目纯 JS，保持一致；以后切 TS 再统一改）
