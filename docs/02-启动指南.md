# 02 · 启动指南

> 杭州医学院 门诊 HIS 系统 · 实训文档

## 启动后端

```bash
cd backend

# Linux/Mac
./gradlew bootRun

# Windows
gradlew.bat bootRun
```

启动成功后：

- 接口地址：`http://localhost:8080/api/...`
- Swagger 文档：`http://localhost:8080/swagger-ui.html`
- 默认账号会在第一次启动时自动初始化（控制台日志中会打印一行：`初始化默认账号：admin / reception / doctor`）

### 配置项

后端启动配置位于：

- `backend/src/main/resources/application.yaml` —— 公共配置（端口、Sa-Token、MyBatis）
- `backend/src/main/resources/application-dev.yaml` —— 开发环境数据库
- `backend/src/main/resources/application-test.yaml` —— 测试环境数据库（默认连接 `his_test` 库）

## 启动前端

```bash
cd frontend
pnpm install        # 第一次需要安装依赖
pnpm dev
```

浏览器访问 `http://localhost:5173`，使用以下账号登录：

| 用户名 | 密码 | 推荐先体验 |
| --- | --- | --- |
| `admin` | 123456 | 看全部菜单 |
| `reception` | 123456 | 患者建档 + 挂号 |
| `doctor` | 123456 | 待诊队列 + 接诊开方 |

> 前端开发服务器已在 `vite.config.js` 中配置好 `/api` 代理到后端 `http://localhost:8080`，无需额外处理跨域。

## 常见问题

**启动后端报错 `Access denied for user 'root'@'localhost'`**

修改 `application-dev.yaml` 里的数据库账号密码。

**前端登录请求 401**

通常是后端没启动或代理没生效，确认浏览器 Network 面板请求是否打到 `localhost:5173/api/...`。

**端口被占用**

后端：修改 `application.yaml` 中的 `server.port`。
前端：在 `vite.config.js` 修改 `server.port`，并同步修改 `proxy.target`。
