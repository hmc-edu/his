# 02 · 启动指南

> 杭州医学院 门诊 HIS 系统 · 实训文档

本文档详细说明三种把系统跑起来的方式，以及每一步**预期看到的输出**和**遇到错误怎么办**。

---

## 三种启动方式怎么选

| 方式 | 适合谁 | 优点 | 缺点 |
| --- | --- | --- | --- |
| **A. Docker Compose 一键启动** | 只想看效果，不改代码 | 一条命令，自带 MySQL；国内镜像配好 | 第一次启动慢；改了代码要重新 build |
| **B. 本地开发模式（全部装到本机）** | 不愿意装 Docker 的同学 | 不依赖 Docker | 要装 JDK / Node / MySQL |
| **C. 混合模式（推荐改代码用）** ⭐ | 学生改代码、做扩展练习 | Docker 起 MySQL，IDEA 起后端可断点；前端 vite 热更 | 要装 JDK / Node + Docker |

---

## A. Docker Compose 一键启动

### 前置条件

装好 Docker Desktop（见 [01-环境准备 §7](./01-环境准备.md#7-docker-desktop可选)）。

### 启动

```bash
cd his
docker compose up --build
```

项目内 `docker-compose.yml` 已经把 mysql / openjdk / node / nginx 全部走 `m.daocloud.io` 镜像，**国内网络可直接拉，不需要额外配 Docker Hub 加速**。已在杭州医学院校园网验证。

### 启动过程做了什么

第一次会按顺序：

1. 拉取基础镜像 `mysql:8.0`、`eclipse-temurin:17-jdk-alpine`、`node:20-alpine`、`nginx:1.25-alpine`（约 200–500 MB，全部走 daocloud 中转）
2. 在容器内编译后端（`./gradlew bootJar`）和前端（`pnpm build`）
3. 启动 MySQL → 跑 `schema.sql` 建表 → 跑 `data.sql` 初始化字典数据
4. 启动后端（连上 MySQL，DataInitializer 创建 admin/reception/doctor 账号）
5. 启动 nginx 前端（80 端口，映射到宿主机 8000）

总耗时：5–15 分钟（取决于网速）。

### 看到这些日志说明启动成功

```
his-mysql    | [System] [MY-013606] [Server] /usr/sbin/mysqld: ready for connections.
his-backend  | Started HisApplication in 8.234 seconds (process running for 9.123)
his-backend  | 初始化默认账号：admin / reception / doctor，密码均为 123456
his-frontend | nginx: configuration file /etc/nginx/nginx.conf test is successful
```

### 验证

打开浏览器：

- 前端：<http://localhost:8000>，应看到登录页
- API 文档：<http://localhost:8080/swagger-ui.html>，应看到 Swagger UI

用 `admin / 123456` 登录前端，看到主菜单即成功。患者列表里"张三 / 男 / 北京市东城区东长安街 1 号"应该正确显示中文。

### 常用操作

```bash
# 后台运行
docker compose up -d --build

# 查看实时日志
docker compose logs -f
docker compose logs -f backend     # 单独看后端

# 停掉（保留数据）
docker compose down

# ⚠️ 停掉并清空数据库（数据混乱 / 中文乱码 / 改了 schema 时用）
docker compose down -v

# 进入 MySQL 容器看数据（不需要本地装 mysql 客户端）
docker exec -it his-mysql mysql -uroot -proot --default-character-set=utf8mb4 his
```

## B. 本地开发模式

适合要改代码、做实训作业、断点调试的同学。

### 前置条件

JDK 17、Node 18+、pnpm、MySQL 全部装好（见 [01-环境准备](./01-环境准备.md)）。

### 第一步：初始化数据库

```bash
mysql -uroot -proot
```

进入后：

```sql
CREATE DATABASE IF NOT EXISTS his DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
USE his;
SOURCE /项目绝对路径/backend/src/main/resources/db/schema.sql;
SOURCE /项目绝对路径/backend/src/main/resources/db/data.sql;
SHOW TABLES;        -- 应看到 8 张表
EXIT;
```

> Windows 下 SOURCE 用正斜杠路径，例如 `SOURCE D:/code/his/backend/src/main/resources/db/schema.sql;`

### 第二步：启动后端

```bash
cd his/backend
./gradlew bootRun     # Linux/Mac
gradlew.bat bootRun   # Windows
```

第一次会下载 Gradle 8.5 + 全部依赖，3–8 分钟。

后端在端口 **8080** 上监听。

#### 验证后端

打开 <http://localhost:8080/swagger-ui.html>，应看到 Swagger UI 列出所有 API。

也可以用 `curl` 调一次登录：

```bash
curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"123456"}'
```

应返回类似：

```json

  "msg": "ok",

    "tokenValue": "abcd1234-...",
    "userInfo": {"id":1,"username":"admin","realName":"系统管理员","role":"ADMIN"}

```

### 第三步：启动前端

**新开一个终端**（不要把后端那个关了）：

```bash
cd his/frontend
pnpm install   # 第一次需要安装依赖，2 分钟
pnpm dev
```

打开 <http://localhost:5173>，应看到登录页（蓝绿色渐变背景）。
用 `admin / 123456` 登录。

## C. 混合模式（推荐做扩展练习的同学用）⭐

最舒服的开发组合：**Docker 起 MySQL，IDEA 起后端，终端 `pnpm dev` 起前端**。这样既不用本机装 MySQL，又能在 IDEA 里打断点调试后端。

### 前置条件

JDK 17、Node 18+、pnpm、Docker Desktop（不需要本机装 MySQL）。

### 第一步：只起 MySQL 容器

```bash
cd his
docker compose up -d mysql
```

`-d` 表示后台运行。MySQL 第一次启动会跑 `schema.sql` 和 `data.sql` 完成建表与字典初始化，需要 5–15 秒。

确认 ready：

```bash
docker compose ps mysql
# STATUS 列出现 (healthy) 才算就绪

# 或直接跟日志，看到 "ready for connections" 即可
docker compose logs -f mysql
```

### 第二步：IDEA 启动后端

`backend/src/main/resources/application-dev.yaml` 默认连接 `localhost:3306`，正好命中刚才映射出来的 mysql 容器端口，**无需改任何配置**。

IDEA 里打开 `HisApplication.java`，点 main 方法旁的绿三角 → `Run 'HisApplication'`（或虫子图标 Debug 模式启动以便打断点）。

控制台出现：

```
Started HisApplication in 5.234 seconds
初始化默认账号：admin / reception / doctor，密码均为 123456
```

后端就 ready 了。

### 第三步：另开终端启前端

```bash
cd his/frontend
pnpm install   # 第一次
pnpm dev
```

打开 <http://localhost:5173> 登录测试。

### 几个常用变种

```bash
# 只想验证前端，不改后端 → docker 起 mysql + backend 一起
docker compose up -d mysql backend
cd frontend && pnpm dev
# 此时 IDEA 不用启后端，前端走容器里的后端

# 反过来：只起 mysql，前端也走 docker，后端 IDEA 调试
# ❌ 不推荐：容器里的前端 nginx 写死了反代 backend:8080，
#    没法通过容器网络访问宿主机 IDEA 里的后端
# 推荐还是走标准混合模式：docker 起 mysql，IDEA 起后端，终端 pnpm dev 起前端
```

### 常用辅助命令

```bash
docker compose ps                        # 看哪些 service 在跑
docker compose stop mysql                # 停 mysql 但保留容器和数据
docker compose start mysql               # 再启动
docker compose restart backend           # 重启容器化的后端
docker compose down                      # 全部停掉并删容器（数据 volume 保留）
docker compose down -v                   # ⚠️ 连 volume 一起删（重置数据库）

# 直接进 mysql 容器看数据
docker exec -it his-mysql mysql -uroot -proot --default-character-set=utf8mb4 his
```

### IDEA 里如何让后端连容器 mysql

容器把 mysql 端口映射到了宿主机 `localhost:3306`，所以 IDEA 启动的后端**不需要任何改动**就能连上——`application-dev.yaml` 里的 `jdbc:mysql://localhost:3306/his?...` 直接生效。

如果想在 IDEA 里查看 mysql 数据，可以装 IDEA 自带的 Database 工具：

1. 右侧栏 Database 图标 → `+` → `Data Source` → `MySQL`
2. 填：Host `localhost`、Port `3306`、Database `his`、User `root`、Password `root`
3. Test Connection → OK → Apply

之后在 IDEA 里就能直接跑 SQL、看表结构。

## 配置项速查

### 改后端端口 / 数据库

修改 `backend/src/main/resources/application.yaml`（端口）和 `application-dev.yaml`（数据库）：

```yaml
# application.yaml
server:
  port: 8080         # ← 改端口

# application-dev.yaml
spring:
  datasource:
    username: root   # ← MySQL 用户名
    password: root   # ← MySQL 密码
    url: jdbc:mysql://localhost:3306/his?...
```

如果你改了后端端口，前端 `frontend/vite.config.js` 里的代理目标也要改：

```js
proxy: {
  '/api': {
    target: 'http://localhost:8080',  // ← 这里
  }
}
```

### 改前端端口

修改 `frontend/vite.config.js`：

```js
server: {
  port: 5173,    // ← 改端口
}
```

## 常见启动错误

| 现象 | 原因 | 解决 |
| --- | --- | --- |
| `Web server failed to start. Port 8080 was already in use` | 端口冲突 | 关掉占用端口的进程，或改 `server.port` |
| `Communications link failure` 启动后端时 | MySQL 没启动 / 端口不对 | 启动 MySQL 服务；确认 3306 端口 |
| `Access denied for user 'root'@'localhost'` | MySQL 密码不对 | 改 `application-dev.yaml` 或 MySQL 密码 |
| `Unknown database 'his'` | 没建库 | 执行 `CREATE DATABASE his ...` |
| `Table 'his.sys_user' doesn't exist` | 没跑 schema.sql | `mysql -uroot -proot his < backend/src/main/resources/db/schema.sql` |
| 前端 `proxy error: ECONNREFUSED` | 后端没启动 | 先启动后端，等到 `Started HisApplication` 再开前端 |
| 前端登录 401 | 后端起来了但 token 不对 | 浏览器 F12 看 Network 面板，确认 Authorization 头被带上 |
| `pnpm: command not found` | 没装 pnpm | `npm install -g pnpm` |
| `gradle: Could not GET ...` | 网络问题 | 见 [05-国内网络优化](./05-国内网络优化.md) |
| `error getting credentials - err: exit status 1` | docker 凭证 helper 没装 | `echo '{}' > ~/.docker/config.json` |
| 中文显示为 `?` 或乱码 | volume 没清掉 | `docker compose down -v && docker compose up --build` |

更多错误见 [11-常见错误-FAQ.md](./11-常见错误-FAQ.md)。
