## 2. 命名规范（很重要，请严格遵守）

| 元素 | 规范 | 例子 |
| --- | --- | --- |
| 包名 | 全小写，按层划分 | `com.hmc.his.service.impl` |
| 类名 | 大驼峰，名词或名词词组 | `Patient`、`PatientService`、`PatientServiceImpl` |
| 接口实现类 | 接口名 + `Impl` | `PatientServiceImpl` |
| Controller | 业务模型 + `Controller` | `PatientController`，`@RequestMapping("/api/patients")` |
| Service | 业务模型 + `Service` | `PatientService` |
| Repository | 业务模型 + `Repository` | `PatientRepository`（不叫 `Mapper` 或 `Dao`） |
| Model（PO） | 业务模型名 | `Patient` 对应表 `patient` |
| DTO | 用途 + `Req` / `Res` | `LoginReq`、`PageRes` |
| 方法名 | 小驼峰，动词开头 | `selectById`、`createRegistration`、`finish` |
| 数据库字段 | 全小写下划线 | `created_at`、`reg_no`、`patient_id` |
| Java 字段 | 小驼峰 | `createdAt`、`regNo`、`patientId` |

> **下划线 ↔ 驼峰转换**由 MyBatis 的 `map-underscore-to-camel-case: true` 自动完成（已在 `application.yaml` 配置）。所以你 SQL 里写 `reg_no`，Model 里就写 `private String regNo;`，无需手工映射。

## 3. 每一层应该写什么、不写什么

### 3.1 Controller 层

✅ **应该做**：
- 接收 HTTP 请求（`@GetMapping` / `@PostMapping` 等）
- 校验请求参数（`@Valid`、`@RequestParam` 默认值）
- 鉴权（`@SaCheckRole("ADMIN")`）
- 调用 Service
- 把 Service 返回值包成 `R<T>` 返回

❌ **不应该做**：
- 写 SQL（那是 Repository 的事）
- 写复杂业务逻辑（那是 Service 的事）
- 直接抛 SQLException、JDBC 异常

**模板**：

```java
@Tag(name = "患者")
@RestController
@RequestMapping("/api/patients")
@RequiredArgsConstructor
public class PatientController {

    private final PatientService patientService;       // 用构造注入，不要 @Autowired 字段

    @GetMapping
    public R<PageRes<Patient>> page(
            @RequestParam(required = false) String keyword,
            @RequestParam(defaultValue = "1") int page,
            @RequestParam(defaultValue = "10") int size) {
        return R.ok(patientService.page(keyword, page, size));
    }

    @PostMapping
    public R<Long> create(@RequestBody @Valid PatientCreateReq req) {
        return R.ok(patientService.create(req));
    }
}
```

> 注意：上面 `PatientCreateReq` 是个 DTO，本项目里 Patient 模块没有专门拆 DTO（直接传 Model），但**当字段差异大时一定要拆 DTO**（参见 `RegistrationCreateReq`）。

### 3.2 Service 层

✅ **应该做**：
- 业务规则（"挂号单状态必须是 WAITING 才能取消"）
- 跨表/跨 Repository 协作
- 事务边界（`@Transactional`）
- 业务异常（`throw new BusinessException("...")`）

❌ **不应该做**：
- 直接写 SQL
- 直接处理 HTTP 相关的对象（HttpServletRequest 等）
- 把 PO 直接当成接口契约——用 DTO 解耦

**模板**：

```java
@Service
@RequiredArgsConstructor
public class RegistrationServiceImpl implements RegistrationService {

    private final RegistrationRepository registrationRepository;
    private final PatientService patientService;
    private final DoctorService doctorService;

    @Override
    public Registration create(RegistrationCreateReq req) {
        // 1. 业务校验
        Patient patient = patientService.get(req.getPatientId());     // 不存在会抛异常
        Doctor doctor   = doctorService.get(req.getDoctorId());

        // 2. 装配业务对象
        Registration r = new Registration();
        r.setPatientId(patient.getId());
        r.setDoctorId(doctor.getId());
        r.setDeptId(doctor.getDeptId());
        r.setRegFee(doctor.getRegFee());
        r.setStatus("WAITING");
        r.setRegNo(generateRegNo(LocalDate.now()));

        // 3. 持久化
        registrationRepository.insert(r);

        // 4. 回查（带上 join 字段）后返回
        return registrationRepository.selectById(r.getId());
    }
}
```

### 3.3 Repository 层

✅ **应该做**：
- 写 SQL（注解或 XML）
- 把结果集映射为 Model

❌ **不应该做**：
- 业务逻辑（"挂号单状态必须是…"——属于 Service）
- 跨表组合多个 SQL 来完成一个业务动作（让 Service 多调几次）

**两种写法**：

```java
// 注解写法（DepartmentRepository）
@Mapper
public interface DepartmentRepository {
    @Select("SELECT * FROM department ORDER BY sort ASC, id ASC")
    List<Department> selectAll();

    @Insert("INSERT INTO department(code, name, sort) VALUES(#{code}, #{name}, #{sort})")
    @Options(useGeneratedKeys = true, keyProperty = "id")
    int insert(Department dept);
}
```

```java
// XML 写法（PatientRepository + PatientMapper.xml）
@Mapper
public interface PatientRepository {
    List<Patient> selectPage(@Param("keyword") String keyword,
                             @Param("offset") int offset,
                             @Param("size") int size);
}
```

```xml
<!-- mappers/PatientMapper.xml -->
<select id="selectPage" resultMap="PatientMap">
    SELECT * FROM patient
    <where>
        <if test="keyword != null and keyword != ''">
            AND (name LIKE CONCAT('%', #{keyword}, '%') OR phone LIKE ...)
        </if>
    </where>
    ORDER BY id DESC
    LIMIT #{offset}, #{size}
</select>
```

**何时选哪个？**
- 简单 CRUD（一行 SQL，无动态条件）→ 注解
- 多个动态 `<if>` / `<foreach>` / 复杂 join → XML

### 3.4 Model 层

字段一对一映射数据库列，加 `@Data` 自动生成 getter/setter。**带 join 显示用的字段**（如 `Doctor.deptName`）也写在 Model 里，但**不出现在 INSERT/UPDATE SQL 中**——它们只是 SELECT 时的"附加列"。

```java
@Data
public class Registration {
    private Long id;
    private String regNo;
    private Long patientId;
    // ... 数据库字段

    /** 关联展示字段（非数据库列） */
    private String patientName;     // SELECT 时 join 来的
    private String doctorName;
}
```

### 3.5 DTO 层

```java
// 入参 DTO：只有需要从前端传的字段
@Data
public class RegistrationCreateReq {
    @NotNull(message = "请选择患者")
    private Long patientId;

    @NotNull(message = "请选择医生")
    private Long doctorId;
}

// 响应 DTO：直接用 Model 也可以，但敏感字段（如 password）一定要清掉
```

## 4. 核心机制速查

### 4.1 统一响应 R<T>

```java
return R.ok();                 // {code:0, msg:"ok", data:null}
return R.ok(patient);          // {code:0, msg:"ok", data:{...}}
return R.fail(404, "未找到");  // {code:404, msg:"未找到"}
```

业务异常推荐用 `BusinessException`：

```java
throw new BusinessException("用户不存在");          // code=400
throw new BusinessException(404, "用户不存在");     // 自定义 code
```

`GlobalExceptionHandler` 会统一翻译成 `R`。

### 4.2 分页

```java
public PageRes<Patient> page(String keyword, int page, int size) {
    int offset = Math.max(0, (page - 1) * size);
    List<Patient> records = patientRepository.selectPage(keyword, offset, size);
    long total = patientRepository.countPage(keyword);
    return PageRes.of(total, records);
}
```

### 4.3 鉴权

```java
// 类级：整个 Controller 都要 ADMIN
@RestController @SaCheckRole("ADMIN")
public class UserController { ... }

// 方法级
@SaCheckRole("DOCTOR")
@PostMapping("/{id}/finish")
public R<Void> finish(@PathVariable Long id) { ... }

// 拿当前登录用户
long uid = StpUtil.getLoginIdAsLong();
SysUser u = (SysUser) StpUtil.getSession().get("user");
```

### 4.4 事务

```java
@Service
@RequiredArgsConstructor
public class VisitServiceImpl implements VisitService {

    @Transactional   // 整个方法在一个事务里，任何异常自动回滚
    public Prescription prescribe(Long visitId, PrescriptionCreateReq req) {
        ...
        prescriptionRepository.insert(p);
        for (...) {
            prescriptionRepository.insertItem(item);
        }
        prescriptionRepository.updateTotal(p.getId(), total);
    }
}
```

### 4.5 校验

```java
// DTO 上加注解
public class LoginReq {
    @NotBlank(message = "用户名不能为空")
    private String username;

    @NotBlank(message = "密码不能为空")
    private String password;
}

// Controller 上 @Valid
@PostMapping("/login")
public R<LoginRes> login(@RequestBody @Valid LoginReq req) { ... }
```

校验失败由 `GlobalExceptionHandler#handleValid` 翻译为 `R{code:400, msg:"用户名不能为空"}`。

## 5. 加一个新模块的标准步骤（5 步法）

以"我要加一个 **`exam_order` 检查申请单**"为例（[实战教程](./13-实战-添加检查申请模块.md) 有完整代码，这里只列骨架）：

### Step 1 — 改数据库

`backend/src/main/resources/db/schema.sql` 加表：

```sql
DROP TABLE IF EXISTS exam_order;
CREATE TABLE exam_order (
    id          BIGINT       NOT NULL AUTO_INCREMENT,
    visit_id    BIGINT       NOT NULL,
    item_name   VARCHAR(100) NOT NULL,
    status      VARCHAR(20)  NOT NULL DEFAULT 'PENDING',
    ordered_at  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (id),
    KEY idx_eo_visit (visit_id)
) ENGINE = InnoDB DEFAULT CHARSET = utf8mb4 COMMENT '检查申请';
```

执行 `mysql -uroot -proot his < schema.sql`（或者干脆删库重建）。

### Step 2 — 写 Model

`backend/src/main/java/com/hmc/his/model/ExamOrder.java`：

```java
@Data
public class ExamOrder {
    private Long id;
    private Long visitId;
    private String itemName;
    private String status;
    private LocalDateTime orderedAt;
}
```

### Step 3 — 写 Repository

`backend/src/main/java/com/hmc/his/repository/ExamOrderRepository.java`：

```java
@Mapper
public interface ExamOrderRepository {
    @Select("SELECT * FROM exam_order WHERE visit_id = #{visitId}")
    List<ExamOrder> selectByVisitId(Long visitId);

    @Insert("INSERT INTO exam_order(visit_id, item_name, status) " +
            "VALUES(#{visitId}, #{itemName}, #{status})")
    @Options(useGeneratedKeys = true, keyProperty = "id")
    int insert(ExamOrder o);
}
```

### Step 4 — 写 Service

接口 + Impl 两个文件，照抄 `DepartmentService` / `DepartmentServiceImpl` 的结构。

### Step 5 — 写 Controller

```java
@Tag(name = "检查申请")
@RestController
@RequestMapping("/api/exam-orders")
@RequiredArgsConstructor
public class ExamOrderController {
    private final ExamOrderService examOrderService;

    @GetMapping
    public R<List<ExamOrder>> list(@RequestParam Long visitId) {
        return R.ok(examOrderService.list(visitId));
    }

    @PostMapping
    public R<Long> create(@RequestBody ExamOrder order) {
        return R.ok(examOrderService.create(order));
    }
}
```

启动后端，访问 <http://localhost:8080/swagger-ui.html> 应能看到新接口。详细操作见 [13-实战教程](./13-实战-添加检查申请模块.md)。

## 6. 调试技巧

### 6.1 看 SQL 实际输出了什么

`application-dev.yaml` 已经配了：

```yaml
logging:
  level:
    com.hmc.his.repository: debug
```

启动后控制台会打印每条 SQL（带参数）。

### 6.2 加日志

```java
@Slf4j      // Lombok 注解，自动生成 log 字段
@Service
public class PatientServiceImpl {

    public Patient get(Long id) {
        log.debug("查询患者 id={}", id);     // {} 是占位符，比 + 拼接快
        Patient p = patientRepository.selectById(id);
        log.info("查询结果: {}", p);
        return p;
    }
}
```

日志级别从低到高：`trace < debug < info < warn < error`。生产环境一般只开 info，开发可以开 debug。

### 6.3 IDEA 断点调试

详见 [10-IDE-与调试-Git.md](./10-IDE-与调试-Git.md)。

### 6.4 直接调接口

不用前端，用 Swagger UI 或 curl：

```bash
# 登录拿 token
TOKEN=$(curl -s -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"123456"}' | jq -r .data.tokenValue)

# 带 token 调接口
curl -H "Authorization: $TOKEN" http://localhost:8080/api/patients
```

## 7. 常见后端"坑"清单

| 坑 | 现象 | 解决 |
| --- | --- | --- |
| 忘了 `@RestController` | 接口返回 404 | 类上加 `@RestController` |
| 忘了 `@Mapper` | 启动报 `Field xxxRepository in xxxServiceImpl required a bean` | 接口加 `@Mapper` 注解 |
| XML 没生效 | `Invalid bound statement (not found): xxx.selectXxx` | 检查 `mappers/*.xml` 路径、namespace 与接口全限定名一致 |
| Model 字段值始终为 null | SELECT 出来的 column 名与 Model 字段名对不上 | 注意 snake/camel 转换；XML 用 `<resultMap>` |
| `@Transactional` 不生效 | 异常没回滚 | `@Transactional` 不能加在 private 方法上、不能从同一个类内方法直接调用（要走代理）|
| Sa-Token 报 `NotLoginException` 但前端明明登录了 | token 没带 / 名字不对 | F12 看请求头，确认 `Authorization` 里有 token |
| 改了代码不生效 | bootRun 是 watch 模式但没自动重启 | IDEA 改完按 Ctrl+F9 重新编译；或装 `spring-boot-devtools` |
