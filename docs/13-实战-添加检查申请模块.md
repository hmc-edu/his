# 13 · 实战 · 从零加一个"检查申请"模块

> 杭州医学院 门诊 HIS 系统 · 实训文档

本文档手把手教你**端到端**地新加一个完整模块：从建表到 Repository、Service、Controller，再到前端 API、页面、路由、菜单。

完成后，医生在就诊页能给患者开"检查申请单"（如胸部 X 光、血常规），技师可以查看并标记完成。

读完 [06-架构总览](./06-架构总览.md)、[07-后端开发指南](./07-后端开发指南.md)、[08-前端开发指南](./08-前端开发指南.md) 再来做这个练习，效果最好。

## 0. 需求分析

### 0.1 业务目标

医生看完病人后，除了开处方外，常常要让患者去做检查（X 光、心电图、CT 等）或检验（血常规、尿常规等）。本模块只做**检查申请**这一类（检验类似，作为延伸练习）。

### 0.2 角色与操作

| 角色 | 能做什么 |
| --- | --- |
| DOCTOR | 在就诊页 / 工作站新增检查申请；查看自己开的申请 |
| RECEPTION | 不参与（这个流程在医生工作站完成） |
| ADMIN | 全部权限 + 字典维护 |
| (新增) TECHNICIAN | 看待执行的申请；标记完成；填检查结果 |

为了简化，本教程暂时**不引入 TECHNICIAN 角色**，让 DOCTOR/ADMIN 自己完成"标记完成"。引入新角色作为扩展。

### 0.3 数据模型

```
visit (就诊单)
  │
  │ 1:N
  ▼
exam_order (检查申请)
  • id
  • visit_id            ── 关联哪个就诊
  • item_name           ── 检查项目名称（"胸部 X 光"）
  • item_code           ── 检查项目代码（"XRAY_CHEST"）
  • status              ── 状态机
  • result              ── 检查结果（完成时填）
  • ordered_at          ── 开单时间
  • completed_at        ── 完成时间
```

状态机：

```
ORDERED ─────▶ COMPLETED
   │
   └─────▶ CANCELLED
```

### 0.4 接口设计

| 方法 | 路径 | 谁能调 | 干什么 |
| --- | --- | --- | --- |
| GET | `/api/exam-orders?visitId=` | 登录 | 看某次就诊的全部检查申请 |
| GET | `/api/exam-orders/{id}` | 登录 | 详情 |
| POST | `/api/exam-orders` | 登录 | 新增 |
| POST | `/api/exam-orders/{id}/complete` | 登录 | 标记完成 + 填结果 |
| POST | `/api/exam-orders/{id}/cancel` | 登录 | 取消申请 |

## 1. 改数据库

### 1.1 加表

打开 `backend/src/main/resources/db/schema.sql`，在末尾追加：

```sql
-- ----------------------------
-- exam_order 检查申请
-- ----------------------------
DROP TABLE IF EXISTS exam_order;
CREATE TABLE exam_order (
    id            BIGINT       NOT NULL AUTO_INCREMENT,
    visit_id      BIGINT       NOT NULL,
    item_code     VARCHAR(50)  NOT NULL COMMENT '检查项目代码',
    item_name     VARCHAR(100) NOT NULL COMMENT '检查项目名称',
    status        VARCHAR(20)  NOT NULL DEFAULT 'ORDERED' COMMENT 'ORDERED/COMPLETED/CANCELLED',
    result        VARCHAR(1000) NULL COMMENT '检查结果',
    ordered_at    DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
    completed_at  DATETIME     NULL,
    created_at    DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at    DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    PRIMARY KEY (id),
    KEY idx_eo_visit (visit_id),
    KEY idx_eo_status (status)
) ENGINE = InnoDB DEFAULT CHARSET = utf8mb4 COMMENT '检查申请';
```

> 命名注意：检查项目（item_code/item_name）将来可以独立成 `exam_item` 字典表，目前直接存在 `exam_order` 里，足够教学。

```java
package com.hmc.his.model;

import lombok.Data;

import java.time.LocalDateTime;

@Data
public class ExamOrder {
    private Long id;
    private Long visitId;
    private String itemCode;
    private String itemName;
    /** ORDERED / COMPLETED / CANCELLED */
    private String status;
    private String result;
    private LocalDateTime orderedAt;
    private LocalDateTime completedAt;
    private LocalDateTime createdAt;
    private LocalDateTime updatedAt;
}
```

要点：
- `@Data` 来自 Lombok，自动生成 getter/setter
- 字段名小驼峰，对应数据库 `item_code` ↔ `itemCode`（自动转）
- 时间用 `LocalDateTime`（不要用 `Date`）

## 3. 后端：Repository

新建 `backend/src/main/java/com/hmc/his/repository/ExamOrderRepository.java`：

```java
package com.hmc.his.repository;

import com.hmc.his.model.ExamOrder;
import org.apache.ibatis.annotations.*;

import java.util.List;

@Mapper
public interface ExamOrderRepository {

    @Select("SELECT * FROM exam_order WHERE visit_id = #{visitId} ORDER BY id DESC")
    List<ExamOrder> selectByVisitId(Long visitId);

    @Select("SELECT * FROM exam_order WHERE id = #{id}")
    ExamOrder selectById(Long id);

    @Insert("INSERT INTO exam_order(visit_id, item_code, item_name, status, ordered_at) " +
            "VALUES(#{visitId}, #{itemCode}, #{itemName}, #{status}, #{orderedAt})")
    @Options(useGeneratedKeys = true, keyProperty = "id")
    int insert(ExamOrder o);

    @Update("UPDATE exam_order SET status = #{status}, result = #{result}, " +
            "completed_at = #{completedAt} WHERE id = #{id}")
    int update(ExamOrder o);

    @Update("UPDATE exam_order SET status = #{status} WHERE id = #{id}")
    int updateStatus(@Param("id") Long id, @Param("status") String status);
}
```

要点：
- `@Mapper` 是 MyBatis 注解，让 Spring 把这个接口注册成可注入的 Bean
- 注解里的 `#{xxx}` 会被自动绑定到方法参数（如果只有一个参数）或字段（如果是 Model）
- `@Options(useGeneratedKeys = true, keyProperty = "id")` 让 INSERT 后把自增 id 回写到对象

## 6. 后端：Service 实现

`backend/src/main/java/com/hmc/his/service/impl/ExamOrderServiceImpl.java`：

```java
package com.hmc.his.service.impl;

import com.hmc.his.common.BusinessException;
import com.hmc.his.dto.ExamOrderCompleteReq;
import com.hmc.his.dto.ExamOrderCreateReq;
import com.hmc.his.model.ExamOrder;
import com.hmc.his.repository.ExamOrderRepository;
import com.hmc.his.service.ExamOrderService;
import lombok.RequiredArgsConstructor;
import org.springframework.stereotype.Service;

import java.time.LocalDateTime;
import java.util.List;

@Service
@RequiredArgsConstructor
public class ExamOrderServiceImpl implements ExamOrderService {

    private final ExamOrderRepository examOrderRepository;

    @Override
    public List<ExamOrder> listByVisit(Long visitId) {
        return examOrderRepository.selectByVisitId(visitId);
    }

    @Override
    public ExamOrder get(Long id) {
        ExamOrder o = examOrderRepository.selectById(id);
        if (o == null) throw new BusinessException("检查申请不存在");
        return o;
    }

    @Override
    public Long create(ExamOrderCreateReq req) {
        ExamOrder o = new ExamOrder();
        o.setVisitId(req.getVisitId());
        o.setItemCode(req.getItemCode());
        o.setItemName(req.getItemName());
        o.setStatus("ORDERED");
        o.setOrderedAt(LocalDateTime.now());
        examOrderRepository.insert(o);
        return o.getId();
    }

    @Override
    public void complete(Long id, ExamOrderCompleteReq req) {
        ExamOrder o = get(id);
        if (!"ORDERED".equals(o.getStatus())) {
            throw new BusinessException("仅已下单状态可完成");
        }
        o.setStatus("COMPLETED");
        o.setResult(req.getResult());
        o.setCompletedAt(LocalDateTime.now());
        examOrderRepository.update(o);
    }

    @Override
    public void cancel(Long id) {
        ExamOrder o = get(id);
        if (!"ORDERED".equals(o.getStatus())) {
            throw new BusinessException("仅已下单状态可取消");
        }
        examOrderRepository.updateStatus(id, "CANCELLED");
    }
}
```

要点：
- `@Service` 让 Spring 把这个类注册成 Service Bean
- `@RequiredArgsConstructor` 自动给 final 字段生成构造器（构造器注入比 `@Autowired` 字段注入更好）
- 状态机收口在 service 层：`complete` / `cancel` 都要先校验当前状态
- 业务异常用 `BusinessException`，`GlobalExceptionHandler` 会接住翻译成 R{code:400, msg:...}

## 7. 后端：Controller

`backend/src/main/java/com/hmc/his/controller/ExamOrderController.java`：

```java
package com.hmc.his.controller;

import com.hmc.his.common.R;
import com.hmc.his.dto.ExamOrderCompleteReq;
import com.hmc.his.dto.ExamOrderCreateReq;
import com.hmc.his.model.ExamOrder;
import com.hmc.his.service.ExamOrderService;
import io.swagger.v3.oas.annotations.tags.Tag;
import jakarta.validation.Valid;
import lombok.RequiredArgsConstructor;
import org.springframework.web.bind.annotation.*;

import java.util.List;

@Tag(name = "检查申请")
@RestController
@RequestMapping("/api/exam-orders")
@RequiredArgsConstructor
public class ExamOrderController {

    private final ExamOrderService examOrderService;

    @GetMapping
    public R<List<ExamOrder>> list(@RequestParam Long visitId) {
        return R.ok(examOrderService.listByVisit(visitId));
    }

    @GetMapping("/{id}")
    public R<ExamOrder> get(@PathVariable Long id) {
        return R.ok(examOrderService.get(id));
    }

    @PostMapping
    public R<Long> create(@RequestBody @Valid ExamOrderCreateReq req) {
        return R.ok(examOrderService.create(req));
    }

    @PostMapping("/{id}/complete")
    public R<Void> complete(@PathVariable Long id, @RequestBody @Valid ExamOrderCompleteReq req) {
        examOrderService.complete(id, req);
        return R.ok();
    }

    @PostMapping("/{id}/cancel")
    public R<Void> cancel(@PathVariable Long id) {
        examOrderService.cancel(id);
        return R.ok();
    }
}
```

```vue
      </template>
    </el-dialog>

    <!-- 完成弹窗 -->
    <el-dialog v-model="completeVisible" title="填写检查结果" width="480px">
      <el-form :model="completeForm" label-width="100px">

        </el-form-item>

        <el-button @click="completeVisible = false">取消</el-button>

<script setup>
import { onMounted, reactive, ref } from 'vue'
import { useRoute } from 'vue-router'
import { ElMessage } from 'element-plus'
import {
  listExamOrdersByVisit, createExamOrder, completeExamOrder, cancelExamOrder
} from '@/api/examOrder'

const route = useRoute()
const visitId = ref(Number(route.params.visitId))

const records = ref([])

const createVisible = ref(false)
const form = reactive({ itemCode: '', itemName: '' })

const completeVisible = ref(false)
const completeForm = reactive({ id: null, result: '' })

const statusOptions = [
  { value: 'ORDERED',   label: '已下单' },
  { value: 'COMPLETED', label: '已完成' },
  { value: 'CANCELLED', label: '已取消' }
]
function statusLabel(s) { return (statusOptions.find(o => o.value === s) || {}).label || s }
function statusType(s) {
  return ({ ORDERED: 'warning', COMPLETED: 'success', CANCELLED: 'info' })[s]
}

async function load() {
  records.value = await listExamOrdersByVisit(visitId.value)
}

function openCreate() {
  Object.assign(form, { itemCode: '', itemName: '' })
  createVisible.value = true
}
async function onCreate() {
  if (!form.itemCode || !form.itemName) return ElMessage.warning('请填写完整')
  await createExamOrder({ visitId: visitId.value, ...form })
  ElMessage.success('已提交')
  createVisible.value = false
  load()
}

function openComplete(row) {
  Object.assign(completeForm, { id: row.id, result: '' })
  completeVisible.value = true
}
async function onComplete() {
  if (!completeForm.result) return ElMessage.warning('请填写检查结果')
  await completeExamOrder(completeForm.id, { result: completeForm.result })
  ElMessage.success('已完成')
  completeVisible.value = false
  load()
}

async function onCancel(row) {
  await cancelExamOrder(row.id)
  ElMessage.success('已取消')
  load()
}

onMounted(load)
</script>
```

## 15. 延伸练习

做完本教程后，强烈建议挑战这些扩展（每个都是经典的工程练习）：

1. **加 `exam_item` 字典表**：检查项目独立成表，开单时用下拉选择，而不是手工填代码和名称
2. **加 TECHNICIAN 角色**：技师能看到"待执行"的检查申请，医生只能看自己开的
3. **完成检查时上传图片**：影像类检查需要附 X 光片
4. **检查申请超时未完成提醒**：定时任务扫超过 1 小时未完成的申请
5. **类比做"检验申请"模块**：`lab_order` + `lab_result`

每个练习独立提一个 PR，逐个攻克。
