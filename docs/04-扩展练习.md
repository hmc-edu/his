# 04 · 扩展练习

主线已经闭环，下面的题目按难度递增。每道题都是给学生练手的脚手架式作业 —— 题面给出动机和大致拆解，具体设计与代码留给学生完成。

---

## ⭐ 练习 1：门诊收费

**目标**：处方提交后生成账单，挂号员/收银员可以收费、退费。

要点：

1. 新增表 `bill`、`bill_item`，与 `prescription` 关联（一份处方一张账单）
2. 新增模块 `bill`：`BillController` / `BillService` / `BillRepository`
3. 在 `VisitServiceImpl#prescribe` 调用结束后，自动生成 `bill`（事务内）
4. 前端新增"收费台"页面：列出待收费账单，操作"收费"后状态置 `PAID`

学习目标：跨模块事务、订单状态机。

---

## ⭐ 练习 2：药房发药

**目标**：处方收费后流入发药队列，药房点击"发药"扣减库存。

要点：

1. 在 `prescription` 表加 `dispense_status` 字段（PENDING / DISPENSED）
2. 新增 `DispenseController`，提供"发药队列查询"和"发药"接口
3. 发药时事务内：
   - 更新 prescription.dispense_status
   - 对每个 item 扣减 `drug.stock`，库存不足应抛 `BusinessException`
4. 前端新增"药房"页面

学习目标：事务、悲观更新、库存校验。

---

## ⭐⭐ 练习 3：检查/检验申请

**目标**：医生在就诊页除了开处方，还可以开"检查申请单"（如血常规、X光），技师收到申请后录入结果。

要点：

1. 新增表 `lab_order` / `lab_result`
2. 新增 `lab` 模块和工作站页面（角色 `LAB`）
3. 在 `Visit.vue` 增加"检查申请"区块

学习目标：模块化设计；多人协作流程（医生 → 技师 → 医生）。

---

## ⭐ 练习 4：报表统计

**目标**：在 dashboard 显示今日门诊量、各科收入、最近 7 天挂号趋势。

要点：

1. 新增 `ReportController`，提供 `/api/reports/today`、`/api/reports/dept-income` 等接口
2. 前端 dashboard 集成 ECharts
3. 思考：聚合 SQL 直接查 vs 引入"日报"表批量计算

学习目标：聚合 SQL；前端图表集成。

---

## ⭐⭐ 练习 5：完整 RBAC 权限模型

**目标**：把 `sys_user.role` 单字段升级为标准 RBAC：`role / menu / role_menu / user_role` 多对多。

要点：

1. 数据库改造（迁移现有数据）
2. 后端：`@SaCheckPermission("sys:user:edit")` 方式做接口鉴权
3. 前端：登录后从后端拉菜单树，按权限点动态渲染侧栏

学习目标：RBAC 概念；前后端权限协同。

---

## ⭐⭐ 练习 6：医生排班与号源

**目标**：挂号不能无限挂号 —— 每个医生每天有 N 个号源，挂满即满号。

要点：

1. 新增表 `doctor_schedule(doctor_id, schedule_date, total, used)`
2. 修改 `RegistrationServiceImpl#create`：开启事务，先查/扣减号源，再插入挂号单（注意并发）
3. 前端在挂号页显示剩余号源数

学习目标：并发控制（乐观锁 / 悲观锁 / select for update）。

---

## ⭐ 练习 7：挂号单与处方单打印

**目标**：挂号成功后弹出可打印的小票；处方提交后可打印处方笺。

要点：

1. 纯前端方案：使用 `window.print()` 配合打印样式表
2. 进阶：服务端生成 PDF（lowagie/iText 或 wkhtmltopdf）

学习目标：浏览器打印；样式 print media query。

---

## ⭐ 练习 8：基于 AOP 的审计日志

**目标**：所有写接口（POST/PUT/DELETE）自动落 `sys_log` 表，记录操作人、URL、参数、IP、耗时。

要点：

1. 新增表 `sys_log`
2. 新建 `@Aspect` 切面，环绕匹配 `controller` 包下的方法
3. 使用 `StpUtil.getLoginIdAsLong()` 获取当前用户

学习目标：AOP；切点表达式。
