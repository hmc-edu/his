# 03 · 业务流程

> 杭州医学院 门诊 HIS 系统 · 实训文档

## 主线流程图

```
 ┌──────────────┐  挂号    ┌─────────────────┐  接诊   ┌───────────────┐  完成   ┌────────┐
 │ 患者建档      │ ─────▶  │ 挂号单(WAITING) │ ─────▶ │ 就诊(VISITING) │ ─────▶ │  DONE  │
 └──────────────┘          └─────────────────┘         └───────────────┘         └────────┘
                                  ▲ 退号                       │
                                  │                            ▼
                              CANCELLED                   开具处方（含明细）
```

挂号单的 `status` 字段是状态机的核心，**只能由后端 service 层切换**：

| 当前状态 | 允许转换 | 触发动作 |
| --- | --- | --- |
| WAITING | VISITING | 医生点击"开始接诊" |
| WAITING | CANCELLED | 挂号员/管理员"退号" |
| VISITING | DONE | 医生点击"完成就诊" |
| DONE / CANCELLED | （终态） | - |

## 端到端冒烟用例

### 1. 挂号员侧（reception 账号）

1. 登录后进入"挂号台 → 患者管理"，新建一个患者（如：王五，男，身份证 110xxx）
2. 进入"挂号"页：
   - 左侧搜索 `王五` → 点击命中行选中
   - 右侧选择"内科"，再选"张医生"
   - 点击"确认挂号"，弹出挂号单 `R20260429xxxx`
3. 进入"挂号查询"页，确认状态为 `待诊`

### 2. 医生侧（doctor 账号）

1. 退出，使用 doctor 账号重新登录
2. 进入"医生工作站 → 待诊队列"，看到刚才的挂号单
3. 点击"开始接诊"，跳转到就诊页：
   - 主诉：发热3天
   - 现病史：3天前受凉后出现…
   - 诊断：上呼吸道感染
   - 医嘱：清淡饮食、多饮水
   - 点击"保存病历"
4. 在"处方"区点击"添加药品"加入：
   - 阿莫西林胶囊：用法 口服，频次 一日三次，数量 2，天数 3
   - 感冒灵颗粒：用法 冲服，频次 一日三次，数量 1，天数 3
5. 点击"提交处方"，下方"已开具处方"出现一条记录
6. 点击"完成就诊"，回到队列页（队列中已不再显示该挂号）

### 3. 数据校验

```sql
-- 挂号单状态
SELECT id, reg_no, status FROM registration ORDER BY id DESC LIMIT 1;
-- 期望: status = 'DONE'

-- 就诊记录
SELECT id, registration_id, diagnosis FROM visit ORDER BY id DESC LIMIT 1;

-- 处方与明细
SELECT * FROM prescription ORDER BY id DESC LIMIT 1;
SELECT * FROM prescription_item WHERE prescription_id = <上一条的 id>;
-- 期望: prescription.total_amount = ∑ prescription_item.subtotal
```

### 4. 接口契约验证（Swagger）

打开 `http://localhost:8080/swagger-ui.html`：

1. `POST /api/auth/login` —— body `{username:"admin", password:"123456"}`，记录返回的 `tokenValue`
2. 点击右上角 "Authorize"，把 token 填入（无需带 `Bearer ` 前缀，Sa-Token 默认即裸 token）
3. 调用 `GET /api/auth/me`，应返回当前用户信息
4. 调用任何写操作不带 token 时，应返回 `{"code":401,"msg":"未登录或登录已过期"}`

## 关键设计点回顾

| 点 | 实现位置 |
| --- | --- |
| 统一响应 R | `common/R.java` + `GlobalExceptionHandler` |
| 状态机切换收口 | `RegistrationServiceImpl#changeStatus` 仅限内部调用 |
| 处方金额服务端计算 | `VisitServiceImpl#prescribe`，前端传入数量不传金额 |
| Sa-Token 拦截白名单 | `SaTokenConfig`：登录与 Swagger 路径放行 |
| MyBatis 双写法对照 | `PatientRepository`(XML) vs `DepartmentRepository`(注解) |
