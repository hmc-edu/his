# 05 · 国内网络优化

> 杭州医学院 门诊 HIS 系统 · 实训文档

校园网/家庭宽带访问 GitHub、Docker Hub、npm、gradle 官方源经常很慢甚至超时。本项目已经把可控的源都换成国内镜像，开箱即用。本文档记录默认配置 + 出问题时怎么排查。

---

## 一览：项目已经配好的镜像

| 资源 | 上游 | 项目内默认 |
| --- | --- | --- |
| Docker 基础镜像（mysql / openjdk / node / nginx） | docker.io | **m.daocloud.io/docker.io/library/...**（在 `docker-compose.yml` 与 Dockerfile 的 `BASE_REGISTRY` 默认值） |
| Maven 依赖 | repo.maven.apache.org | 阿里云 `https://maven.aliyun.com/repository/public/`（在 `backend/build.gradle`） |
| Gradle 发行包 | services.gradle.org | 腾讯云 `https://mirrors.cloud.tencent.com/gradle/`（在 `backend/gradle/wrapper/gradle-wrapper.properties`） |
| npm 包 | registry.npmjs.org | 淘宝 npmmirror `https://registry.npmmirror.com/`（在 `frontend/.npmrc`） |

→ `docker compose up --build`、`./gradlew bootRun`、`pnpm install` 跑起来都自动走国内源，**无需额外配置**。

## 1. Docker Hub 加速（可选，作为兜底）

项目已经把 base image 全部改成 `m.daocloud.io/docker.io/library/...`，正常情况下不需要再配 daemon 镜像加速。但如果某天 daocloud 临时抽风、或者你直接 `docker pull mysql:8.0`，daemon 级镜像加速能兜底。

### Linux

编辑（无则创建）`/etc/docker/daemon.json`：

```json
{
  "registry-mirrors": [
    "https://docker.m.daocloud.io",
    "https://docker.1panel.live"
  ]
}
```

```bash
sudo systemctl daemon-reload
sudo systemctl restart docker
```

### Docker Desktop（Windows / macOS）

Settings → Docker Engine → 把 JSON 加上 `registry-mirrors` → Apply & Restart。

### 验证

```bash
docker info | grep -A 5 'Registry Mirrors'
```

## 2. 阿里云 ACR（可选 · 想要稳定校园网速度时）

如果学校已经有阿里云账号，可把镜像同步到 **阿里云容器镜像服务（个人版免费）**。流程：

1. 阿里云控制台 → 容器镜像服务 → 创建命名空间 `hmc-edu`，类型选 **公开**
2. 创建访问凭证（RAM 子用户 + 密码）
3. 在 GitHub 仓库 Settings → Secrets 添加：
   - `ALIYUN_REGISTRY_USERNAME`
   - `ALIYUN_REGISTRY_PASSWORD`
4. 修改 `.github/workflows/release.yml` 的 `docker-images` job，加一段：
   ```yaml
   - name: Log in to Aliyun ACR
     uses: docker/login-action@v3
     with:
       registry: registry.cn-hangzhou.aliyuncs.com
       username: ${{ secrets.ALIYUN_REGISTRY_USERNAME }}
       password: ${{ secrets.ALIYUN_REGISTRY_PASSWORD }}
   ```
   并在 `docker/build-push-action` 的 `tags:` 里追加：
   ```
   registry.cn-hangzhou.aliyuncs.com/hmc-edu/his-backend:${{ steps.meta.outputs.version }}
   registry.cn-hangzhou.aliyuncs.com/hmc-edu/his-backend:latest
   ```
5. 学生在 `docker-compose.yml` 里把 `build: ./...` 替换为：
   ```yaml
   image: registry.cn-hangzhou.aliyuncs.com/hmc-edu/his-backend:latest
   ```

这样可以跳过本地编译，从阿里云 CDN 直接拉镜像，校园网速度通常能到 10MB/s+。

## 3. 排查清单

| 现象 | 原因 | 解决 |
| --- | --- | --- |
| `Could not GET 'https://services.gradle.org/...gradle-8.5-bin.zip'` | gradle wrapper 用了官方源 | 确认 `gradle-wrapper.properties` 里是 `mirrors.cloud.tencent.com` |
| `pnpm install` 卡在 `Progress: resolved 0` | npm 走默认源 | 确认 `frontend/.npmrc` 存在 |
| `docker pull` 一直卡 / 几十 KB/s | daocloud 临时抖动 / Docker Hub 限速 | 按上面 §1 配 daemon `registry-mirrors` |
| `error getting credentials` | docker 凭证 helper 缺失 | `echo '{}' > ~/.docker/config.json` |
| `git clone` 慢 | GitHub 网络 | 用 GitHub 加速（如 `https://gh-proxy.com`）或学校镜像 |

## 4. 校园网完全离线方案（兜底）

如果实在拉不动任何外部资源，让助教/老师在校外环境构建好镜像，然后：

```bash
# 老师机器（先在能联网的地方跑过 docker compose up --build 把镜像构好）
docker save -o his-backend.tar  his-backend:latest
docker save -o his-frontend.tar his-frontend:latest
docker save -o mysql.tar        m.daocloud.io/docker.io/library/mysql:8.0
# 上传到校园网盘 / U 盘
```

```bash
# 学生机器
docker load -i his-backend.tar
docker load -i his-frontend.tar
docker load -i mysql.tar

# 把 docker-compose.yml 中的 build: 段改成对应的 image: 即可，比如
# backend:
#   image: his-backend:latest
docker compose up
```
