# 05 · 国内网络优化

> 杭州医学院 门诊 HIS 系统 · 实训文档

校园网/家庭宽带访问 GitHub、Docker Hub、npm、gradle 官方源经常很慢甚至超时。本项目已经把可控的源都换成国内镜像，剩下两类（Docker Hub、GHCR）需要在你本机做一次配置即可。

---

## 一览：项目已经配好的镜像

| 资源 | 上游 | 项目内默认 |
| --- | --- | --- |
| Maven 依赖 | repo.maven.apache.org | 阿里云 `https://maven.aliyun.com/repository/public/`（在 `backend/build.gradle`） |
| Gradle 发行包 | services.gradle.org | 腾讯云 `https://mirrors.cloud.tencent.com/gradle/`（在 `backend/gradle/wrapper/gradle-wrapper.properties`） |
| npm 包 | registry.npmjs.org | 淘宝 npmmirror `https://registry.npmmirror.com/`（在 `frontend/.npmrc`） |

→ `pnpm install`、`./gradlew bootRun`、`docker compose up --build` 跑起来就走国内源，无需额外配置。

## 还需要你做的两件事

### 1. Docker Hub 加速（影响所有 `docker pull`）

第一次 `docker compose up --build` 会拉 `mysql:8.0`、`eclipse-temurin:17-jdk-alpine`、`node:20-alpine`、`nginx:1.25-alpine`，全部来自 Docker Hub。不配镜像就会卡很久。

#### Linux

编辑（无则创建）`/etc/docker/daemon.json`：

```json
{
  "registry-mirrors": [
    "https://docker.m.daocloud.io",
    "https://docker.1panel.live"
  ]
}
```

然后：

```bash
sudo systemctl daemon-reload
sudo systemctl restart docker
```

#### Docker Desktop（Windows / macOS）

Settings → Docker Engine，把 JSON 里加：

```json
{
  "registry-mirrors": [
    "https://docker.m.daocloud.io",
    "https://docker.1panel.live"
  ]
}
```

点 "Apply & Restart"。

#### 验证

```bash
docker info | grep -A 5 'Registry Mirrors'
docker pull mysql:8.0      # 应在 30s 内完成
```

### 2. GHCR 镜像访问（仅 "拉预编译镜像" 场景需要）

GHCR 在国内可达，但有两个坑：

**坑一：默认私有**。GitHub Actions 推上来的 org 镜像默认是 Private，不登录就 401。管理员需要去：

- <https://github.com/orgs/hmc-edu/packages/container/his-backend/settings>
- <https://github.com/orgs/hmc-edu/packages/container/his-frontend/settings>

"Danger Zone" → Change package visibility → **Public**。一次性，30 秒。

改完 Public 后，下面这两条命令就能用（速度看校园网，1–5 MB/s 不等）：

```bash
docker pull ghcr.io/hmc-edu/his-backend:latest
docker pull ghcr.io/hmc-edu/his-frontend:latest
docker compose -f docker-compose.cn.yml up
```

**坑二：DaoCloud `m.daocloud.io` 有白名单**。错误示例：

```
denied: 🚫 这镜像不在白名单. this image is not in the allowlist.
```

DaoCloud 的公益镜像库要求自定义仓库提交申请，参见
<https://github.com/DaoCloud/public-image-mirror/issues>，提交后 1–3 天审核通过即可使用 `m.daocloud.io/ghcr.io/...` 前缀。

#### 想要更稳的国内速度：阿里云 ACR 容器镜像服务

如果学校已经有阿里云账号，强烈推荐把镜像同步到 **阿里云容器镜像服务（ACR 个人版免费）**，让学生直接从 `registry.cn-hangzhou.aliyuncs.com` 拉。流程：

1. 阿里云控制台 → 容器镜像服务 → 创建命名空间 `hmc-edu`（公开）
2. 创建访问凭证（RAM 子用户 + 密码）
3. 在 GitHub 仓库 Settings → Secrets 添加：
   - `ALIYUN_REGISTRY_USERNAME`
   - `ALIYUN_REGISTRY_PASSWORD`
4. 在 `.github/workflows/release.yml` 的 `docker-images` job 加一段：
   ```yaml
   - name: Log in to Aliyun ACR
     uses: docker/login-action@v3
     with:
       registry: registry.cn-hangzhou.aliyuncs.com
       username: ${{ secrets.ALIYUN_REGISTRY_USERNAME }}
       password: ${{ secrets.ALIYUN_REGISTRY_PASSWORD }}
   ```
   然后在 `docker/build-push-action` 的 `tags:` 里追加：
   ```
   registry.cn-hangzhou.aliyuncs.com/hmc-edu/his-backend:${{ steps.meta.outputs.version }}
   registry.cn-hangzhou.aliyuncs.com/hmc-edu/his-backend:latest
   ```
5. 学生改用：
   ```yaml
   image: registry.cn-hangzhou.aliyuncs.com/hmc-edu/his-backend:latest
   ```

## 排查清单

| 现象 | 原因 | 解决 |
| --- | --- | --- |
| `Could not GET 'https://services.gradle.org/...gradle-8.5-bin.zip'` | gradle wrapper 还在用官方源 | 确认 `gradle-wrapper.properties` 里是 `mirrors.cloud.tencent.com` |
| `pnpm install` 卡在 `Progress: resolved 0` | npm 走默认源 | 确认 `frontend/.npmrc` 存在 |
| `docker pull mysql` 几十 KB/s | Docker Hub 没加速 | 按上文配 `daemon.json` |
| 拉 ghcr.io 401 / `pull access denied` | GHCR 镜像还是私有 | 管理员去 GHCR 包设置改为 Public（见上节） |
| 拉 m.daocloud.io 报 `not in the allowlist` | 自定义仓库未申请 | 提交白名单申请，或改用 ghcr.io / Aliyun ACR |
| `git clone` 慢 | GitHub 网络 | 用 GitHub 加速（如 `https://gh-proxy.com`）或学校镜像 |

## 校园网完全离线方案（兜底）

实在拉不动，让助教/老师在校外环境构建好镜像，然后：

```bash
# 老师机器：
docker save -o his-backend.tar  ghcr.io/hmc-edu/his-backend:v0.1.1
docker save -o his-frontend.tar ghcr.io/hmc-edu/his-frontend:v0.1.1
docker save -o mysql.tar        mysql:8.0
# 上传到校园网盘 / U 盘

# 学生机器：
docker load -i his-backend.tar
docker load -i his-frontend.tar
docker load -i mysql.tar
docker compose -f docker-compose.cn.yml up
```
