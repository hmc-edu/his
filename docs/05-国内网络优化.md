# 05 · 国内网络优化

> 杭州医学院 门诊 HIS 系统 · 实训文档

校园网/家庭宽带访问 GitHub、Docker Hub、npm、gradle 官方源经常很慢甚至超时。本项目已经把可控的源都换成国内镜像，剩下两类（Docker Hub、GHCR）需要在你本机做一次配置即可。

---

## 一览：项目已经配好的镜像

| 资源 | 上游 | 项目内默认 |
| --- | --- | --- |
| Maven 依赖 | repo.maven.apache.org | 阿里云 `https://maven.aliyun.com/repository/public/`（在 `backend/build.gradle`） |
| Gradle 发行包 | services.gradle.org | 腾讯云 `https://mirrors.cloud.tencent.com/gradle/`（在 `backend/gradle/wrapper/gradle-wrapper.properties`） |
| npm 包 | registry.npmjs.org | 淘宝 npmmirror `https://registry.npmmirror.com/`（在 `frontend/.npmrc`） |

→ `pnpm install`、`./gradlew bootRun`、`docker compose up --build` 跑起来就走国内源，无需额外配置。

---

## 还需要你做的两件事

### 1. Docker Hub 加速（影响所有 `docker pull`）

第一次 `docker compose up --build` 会拉 `mysql:8.0`、`eclipse-temurin:17-jdk-alpine`、`node:20-alpine`、`nginx:1.25-alpine`，全部来自 Docker Hub。不配镜像就会卡很久。

#### Linux

编辑（无则创建）`/etc/docker/daemon.json`：

```json
{
  "registry-mirrors": [
    "https://docker.m.daocloud.io",
    "https://docker.1panel.live"
  ]
}
```

然后：

```bash
sudo systemctl daemon-reload
sudo systemctl restart docker
```

#### Docker Desktop（Windows / macOS）

Settings → Docker Engine，把 JSON 里加：

```json
{
  "registry-mirrors": [
    "https://docker.m.daocloud.io",
    "https://docker.1panel.live"
  ]
}
```

点 "Apply & Restart"。

#### 验证

```bash
docker info | grep -A 5 'Registry Mirrors'
docker pull mysql:8.0      # 应在 30s 内完成
```

### 2. GHCR 镜像加速（仅 "拉预编译镜像" 场景需要）

如果你要按 README 的 **B. 直接拉镜像** 路径跑（不本地编译，直接用 CI 产出的镜像），原始命令是：

```bash
docker pull ghcr.io/hmc-edu/his-backend:latest    # 经常很慢
```

换成 DaoCloud 的 GHCR 中转：

```bash
docker pull m.daocloud.io/ghcr.io/hmc-edu/his-backend:latest
docker pull m.daocloud.io/ghcr.io/hmc-edu/his-frontend:latest
```

然后用项目根目录的 `docker-compose.cn.yml` 直接跑：

```bash
docker compose -f docker-compose.cn.yml up
```

`docker-compose.cn.yml` 与 `docker-compose.yml` 的差别：

- `mysql` 镜像 → `m.daocloud.io/docker.io/library/mysql:8.0`
- `backend` / `frontend` 用 `image:` 指向 `m.daocloud.io/ghcr.io/...`，跳过本地编译

## 排查清单

| 现象 | 原因 | 解决 |
| --- | --- | --- |
| `Could not GET 'https://services.gradle.org/...gradle-8.5-bin.zip'` | gradle wrapper 还在用官方源 | 确认 `gradle-wrapper.properties` 里是 `mirrors.cloud.tencent.com` |
| `pnpm install` 卡在 `Progress: resolved 0` | npm 走默认源 | 确认 `frontend/.npmrc` 存在 |
| `docker pull mysql` 几十 KB/s | Docker Hub 没加速 | 按上文配 `daemon.json` |
| `manifest unknown` 拉 ghcr.io | GHCR 镜像还是私有 | 让管理员把包改 Public（参见 README B 节末尾的链接） |
| `git clone` 慢 | GitHub 网络 | 用 GitHub 加速（如 `https://gh-proxy.com`）或学校镜像 |

## 校园网完全离线方案（兜底）

实在拉不动，让助教/老师在校外环境构建好镜像，然后：

```bash
# 老师机器：
docker save -o his-backend.tar  ghcr.io/hmc-edu/his-backend:v0.1.1
docker save -o his-frontend.tar ghcr.io/hmc-edu/his-frontend:v0.1.1
docker save -o mysql.tar        mysql:8.0
# 上传到校园网盘 / U 盘

# 学生机器：
docker load -i his-backend.tar
docker load -i his-frontend.tar
docker load -i mysql.tar
docker compose -f docker-compose.cn.yml up
```
