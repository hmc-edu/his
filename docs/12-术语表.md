# 12 · 术语表

> 杭州医学院 门诊 HIS 系统 · 实训文档

碰到不认识的词时来这里查。按"业务"和"技术"两类。

---

## 一、业务（医院信息化）

| 术语 | 英文 | 一句话解释 |
| --- | --- | --- |
| **HIS** | Hospital Information System | 医院信息系统，覆盖挂号/收费/医嘱/药房/库存等全流程的核心系统 |
| **门诊** | Outpatient | 不需要住院的就诊服务（与"住院"对应） |
| **挂号** | Registration | 患者预约/取号见医生，是门诊业务的入口 |
| **就诊** | Visit / Encounter | 医生与患者面对面看病的过程 |
| **电子病历** | EMR (Electronic Medical Record) | 数字化的病历记录 |
| **主诉** | Chief Complaint | 患者来看病主要因为什么（"发烧 3 天"） |
| **现病史** | Present Illness | 当前疾病的发展过程 |
| **诊断** | Diagnosis | 医生对疾病的判断（"上呼吸道感染"） |
| **医嘱** | Doctor's Advice / Order | 医生开给患者的指令（用药、休息、复诊等） |
| **处方** | Prescription | 医生开具的药品清单（含剂量、频次、天数） |
| **明细** | Item / Detail | 处方里的每一行（一种药一行） |
| **科室** | Department | 内科 / 外科 / 儿科 等 |
| **职称** | Title | 主任医师 / 副主任医师 / 主治医师 / 住院医师 |
| **挂号费** | Registration Fee | 不同职称医生不同价 |
| **退号** | Cancel Registration | 患者临时不来了，挂号取消 |
| **排班** | Schedule / Shift | 医生哪天上哪个班 |
| **号源** | Slot / Quota | 一个医生一天能接多少患者（限号） |
| **缴费 / 收费** | Payment / Charging | 收银台收钱 |
| **退费** | Refund | 退款 |
| **发药** | Dispense / Dispensing | 药房把处方上的药给患者 |
| **库存** | Stock / Inventory | 药品的现存数量 |
| **检查** | Examination | 影像/物理检查（X 光、B 超、心电图） |
| **检验** | Laboratory Test | 体液样本检测（血常规、尿常规） |
| **ICD** | International Classification of Diseases | 国际疾病分类编码（WHO 维护） |
| **三医联动** | —— | 医保/医疗/医药协同 |
| **住院** | Inpatient | 需要床位过夜的诊疗（本项目不涉及） |
| **HIS / EMR / LIS / PACS / RIS** | —— | HIS 是总系统，EMR 是病历，LIS 是检验科，PACS 是影像存储，RIS 是影像申请 |

## 二、技术 · 后端 / Java

| 术语 | 英文 | 一句话解释 |
| --- | --- | --- |
| **JDK** | Java Development Kit | Java 开发工具包，包含编译器和运行环境 |
| **JVM** | Java Virtual Machine | Java 虚拟机，跑 .class 文件的运行时 |
| **JRE** | Java Runtime Environment | 仅运行环境，不含编译器 |
| **Spring Boot** | —— | 让 Spring 应用开箱即用的脚手架 |
| **Spring** | —— | Java 后端最主流的框架（IoC/AOP/MVC...） |
| **IoC** | Inversion of Control | 控制反转：对象由容器创建并注入，而不是 new 出来 |
| **DI** | Dependency Injection | 依赖注入，IoC 的具体实现方式 |
| **Bean** | —— | 被 Spring 容器管理的对象 |
| **AOP** | Aspect-Oriented Programming | 面向切面编程，把横切关注点（日志、事务）剥离出来 |
| **MVC** | Model-View-Controller | 模型-视图-控制器分层架构 |
| **Controller** | —— | 接 HTTP 请求的层 |
| **Service** | —— | 业务逻辑层 |
| **Repository / DAO** | Data Access Object | 数据访问层 |
| **PO** | Persistence Object | 持久化对象，对应数据库一行 |
| **DTO** | Data Transfer Object | 数据传输对象，前后端契约 |
| **VO** | View Object | 给前端展示用的对象 |
| **MyBatis** | —— | 半自动 ORM 框架，自己写 SQL |
| **JPA** | Java Persistence API | 全自动 ORM 标准（Hibernate 是其实现） |
| **Mapper** | —— | MyBatis 中的 SQL 接口（=Repository） |
| **ORM** | Object-Relational Mapping | 对象关系映射，把表行映射成对象 |
| **Lombok** | —— | 用注解自动生成 getter/setter/构造器 |
| **`@Data`** | —— | Lombok 注解，自动生成 getter/setter/toString/equals/hashCode |
| **`@Slf4j`** | —— | Lombok 注解，自动生成 `log` 字段 |
| **`@Autowired` / `@RequiredArgsConstructor`** | —— | 自动注入依赖（项目用后者，构造器注入） |
| **`@Transactional`** | —— | 标注事务边界 |
| **Sa-Token** | —— | 国产轻量鉴权框架 |
| **JWT** | JSON Web Token | 无状态 token 标准，Sa-Token 也支持 |
| **BCrypt** | —— | 密码哈希算法，单向不可逆 |
| **Gradle** | —— | Java 构建工具（项目用），与 Maven 同类 |
| **Maven** | —— | Java 构建工具，仍是国内主流 |
| **bootJar** | —— | Spring Boot 打包成可运行 jar 的 Gradle 任务 |
| **bootRun** | —— | 直接启动 Spring Boot 应用的 Gradle 任务 |
| **OpenAPI / Swagger** | —— | API 文档规范 + UI |
| **REST** | Representational State Transfer | 一种 API 设计风格（GET/POST/PUT/DELETE 对应增删改查） |
| **Endpoint / 接口** | —— | 一个具体的 HTTP 路径 |
| **Status Code** | —— | HTTP 状态码（200=ok, 401=未登录, 404=未找到, 500=服务器错） |

| **Schema** | —— | 数据库结构定义（也指 DB 实例） |
