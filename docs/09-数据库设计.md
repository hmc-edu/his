# 09 · 数据库设计

> 杭州医学院 门诊 HIS 系统 · 实训文档

## 2. 通用约定

所有表都遵循下面这些约定：

| 约定 | 值 |
| --- | --- |
| 引擎 | InnoDB（支持事务） |
| 字符集 | utf8mb4（支持 emoji 与生僻字） |
| 排序规则 | utf8mb4_unicode_ci |
| 主键 | `id BIGINT NOT NULL AUTO_INCREMENT` |
| 时间戳 | 多数业务表都有 `created_at` / `updated_at`，由 MySQL 自动维护 |
| 字段命名 | 全小写 + 下划线（如 `reg_no`、`patient_id`） |
| 状态字段 | 用 VARCHAR + 枚举值（如 `WAITING/VISITING/DONE/CANCELLED`），不用数字 |
| 金额 | `DECIMAL(10,2)`，绝不用 FLOAT/DOUBLE |
| 软删除 | 当前未启用，删除是物理删除（学生扩展可加 `deleted` 字段） |
| 外键 | **不显式建外键**（教学项目，约束放在 service 层），但有 `idx_*` 索引 |

> **为什么不用外键？** 外键虽好，但在教学场景下批量改表/重建数据时频繁报错，反而妨碍学习。生产系统视团队规范决定。

## 3. 系统字典表

### 3.1 sys_user（系统用户）

```sql
sys_user (
    id          BIGINT       PK AUTO_INCREMENT,
    username    VARCHAR(50)  NOT NULL UNIQUE,    -- 登录名
    password    VARCHAR(100) NOT NULL,           -- BCrypt 加密后的密码
    real_name   VARCHAR(50)  NOT NULL,
    role        VARCHAR(20)  NOT NULL,           -- 'ADMIN' / 'RECEPTION' / 'DOCTOR'
    enabled     TINYINT(1)   NOT NULL DEFAULT 1, -- 是否启用
    created_at  DATETIME,
    updated_at  DATETIME
)
```

**重点**：
- 密码字段存 **BCrypt 哈希**（约 60 字符），永远不存明文。代码里用 `BCryptPasswordEncoder` 加密。
- `role` 设计为单字段枚举，简单粗暴；要做完整 RBAC 见 [04-扩展练习](./04-扩展练习.md) 练习 5。
- 默认账号由 `DataInitializer` 在启动时插入（admin/reception/doctor）。

**对应代码**：`model/SysUser.java`、`repository/SysUserRepository.java`

### 3.2 department（科室）

```sql
department (
    id    BIGINT       PK,
    code  VARCHAR(20)  NOT NULL UNIQUE,   -- 'NK' / 'WK' / 'EK'
    name  VARCHAR(50)  NOT NULL,           -- '内科' / '外科' / '儿科'
    sort  INT          NOT NULL DEFAULT 0  -- 排序，越小越靠前
)
```

**重点**：用 `code` 做业务编号，便于打印/对接外部系统；`sort` 决定下拉显示顺序。

### 3.3 doctor（医生）

```sql
doctor (
    id        BIGINT       PK,
    user_id   BIGINT       NULL,            -- 关联 sys_user.id（可空，代表纯展示医生）
    dept_id   BIGINT       NOT NULL,        -- 关联 department.id
    name      VARCHAR(50)  NOT NULL,
    title     VARCHAR(50)  NULL,            -- '主任医师' '副主任医师' '主治医师'
    reg_fee   DECIMAL(10,2) NOT NULL DEFAULT 10.00,  -- 挂号费
    created_at, updated_at,
    KEY idx_doctor_dept (dept_id),
    KEY idx_doctor_user (user_id)
)
```

**重点**：
- `user_id` 是医生**和登录账号的一对一关联**——一个医生有自己的登录账号才能在工作站工作。
- `reg_fee` 写死在医生上而非科室，因为同科室不同职称医生挂号费不同。

### 3.4 drug（药品目录）

```sql
drug (
    id     BIGINT       PK,
    code   VARCHAR(30)  NOT NULL UNIQUE,    -- 药品编码
    name   VARCHAR(100) NOT NULL,
    spec   VARCHAR(100) NULL,                -- 规格，如 '0.25g*24粒'
    unit   VARCHAR(20)  NOT NULL DEFAULT '盒',
    price  DECIMAL(10,2) NOT NULL DEFAULT 0,
    stock  INT          NOT NULL DEFAULT 0   -- 库存（简化版，无最小库存等）
)
```

**重点**：
- `stock` 是简化版库存，没有"批次"、"效期"、"安全库存"等字段。
- 真实 HIS 通常会拆为 `drug`（药品基础信息）+ `drug_stock`（库存流水），见扩展练习 2。

### 3.5 patient（患者档案）

```sql
patient (
    id        BIGINT       PK,
    name      VARCHAR(50)  NOT NULL,
    gender    VARCHAR(10)  NOT NULL DEFAULT '未知',  -- '男'/'女'/'未知'
    birthday  DATE         NULL,
    phone     VARCHAR(20)  NULL,
    id_card   VARCHAR(30)  NULL UNIQUE,              -- 身份证号
    address   VARCHAR(200) NULL,
    created_at, updated_at,
    KEY idx_patient_name (name),
    KEY idx_patient_phone (phone)
)
```

**重点**：
- `id_card` 加 UNIQUE 防止重复建档，但允许 NULL（小儿、外地病人可能没带）。
- 索引 `idx_patient_name/phone` 支撑挂号台快速搜索。

## 4. 业务流水表

### 4.1 registration（挂号单）—— 状态机入口

```sql
registration (
    id          BIGINT       PK,
    reg_no      VARCHAR(30)  NOT NULL UNIQUE,    -- 'R20260429xxxx'
    patient_id  BIGINT       NOT NULL,            -- 关联 patient
    doctor_id   BIGINT       NOT NULL,            -- 关联 doctor
    dept_id     BIGINT       NOT NULL,            -- 关联 department（冗余，加速查询）
    reg_date    DATE         NOT NULL,
    reg_fee     DECIMAL(10,2) NOT NULL,           -- 创建时从 doctor.reg_fee 拷贝（避免医生改价影响历史单）
    status      VARCHAR(20)  NOT NULL DEFAULT 'WAITING',
        -- WAITING   待诊
        -- VISITING  就诊中
        -- DONE      已就诊
        -- CANCELLED 已退号
    created_at, updated_at,
    KEY idx_reg_doctor_status (doctor_id, status),
    KEY idx_reg_date (reg_date)
)
```

**重点**：
- `reg_no` 由 service 层生成：`R + 日期 + 4 位序号`。
- `reg_fee` **冗余存储**：哪怕医生改了挂号费，已生成的挂号单保持当时价格。
- `(doctor_id, status)` 联合索引：医生工作站查"待诊队列"是高频查询。
- 状态机由 `RegistrationServiceImpl` 集中管理，前端不能直接改 `status` 字段。

### 4.2 visit（就诊/电子病历）

```sql
visit (
    id              BIGINT       PK,
    registration_id BIGINT       NOT NULL UNIQUE,  -- 一个挂号单只有一份就诊记录
    chief_complaint VARCHAR(500) NULL,             -- 主诉
    present_illness VARCHAR(1000) NULL,            -- 现病史
    diagnosis       VARCHAR(500) NULL,             -- 诊断
    doctor_advice   VARCHAR(1000) NULL,            -- 医嘱
    visited_at      DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
    created_at, updated_at
)
```

**重点**：
- `registration_id` 加 UNIQUE：与挂号单**严格一对一**。
- "电子病历四要素"是中国 HIS 行业的最简形态，真实系统还会有过敏史、家族史、查体等更多字段。

### 4.3 prescription（处方）

```sql
prescription (
    id            BIGINT       PK,
    visit_id      BIGINT       NOT NULL,           -- 关联 visit
    prescribed_at DATETIME     NOT NULL,
    total_amount  DECIMAL(10,2) NOT NULL DEFAULT 0,
    KEY idx_prescription_visit (visit_id)
)
```

**重点**：
- `visit_id` 不加 UNIQUE：**一次就诊可以开多份处方**（西药 + 中药 + 注射各一份）。
- `total_amount` 由 service 层计算后写入，**不接受前端直接传**。

### 4.4 prescription_item（处方明细）

```sql
prescription_item (
    id              BIGINT       PK,
    prescription_id BIGINT       NOT NULL,
    drug_id         BIGINT       NOT NULL,
    dosage          VARCHAR(50)  NULL,             -- 剂量，如 '0.25g'
    frequency       VARCHAR(50)  NULL,             -- 频次，如 '一日三次'
    days            INT          NOT NULL DEFAULT 1,
    qty             INT          NOT NULL DEFAULT 1, -- 数量（盒/瓶）
    unit_price      DECIMAL(10,2) NOT NULL,         -- 创建时从 drug.price 拷贝
    subtotal        DECIMAL(10,2) NOT NULL,         -- = unit_price * qty
    KEY idx_pi_prescription (prescription_id),
    KEY idx_pi_drug (drug_id)
)
```

**重点**：
- `unit_price`、`subtotal` 都**冗余存储**——药品调价不影响历史处方金额。
- 总和等式：`prescription.total_amount == SUM(prescription_item.subtotal)`，由 service 在事务内保证。

## 5. 状态机：registration.status 完整规则

```
                  挂号
                   │
                   ▼
              ┌──────────┐  退号  ┌──────────────┐
              │ WAITING  │ ─────▶ │  CANCELLED   │
              │ (待诊)   │        │  (终态)       │
              └────┬─────┘        └──────────────┘
                   │
                   │ 医生"开始接诊"
                   ▼
              ┌──────────┐
              │ VISITING │
              │ (就诊中) │
              └────┬─────┘
                   │
                   │ 医生"完成就诊"
                   ▼
              ┌──────────┐
              │   DONE   │
              │  (终态)   │
              └──────────┘
```

允许的转换（其它转换都抛 `BusinessException`）：

| 当前状态 | 触发动作 | 下一状态 | 触发位置 |
| --- | --- | --- | --- |
| WAITING | 退号 | CANCELLED | `RegistrationService#cancel` |
| WAITING | 接诊 | VISITING | `VisitService#start` |
| VISITING | 完成 | DONE | `VisitService#finish` |
| DONE / CANCELLED | 任何 | （不允许） | 抛异常 |

## 6. 索引设计

| 表 | 索引 | 支撑的查询 |
| --- | --- | --- |
| sys_user | UNIQUE(username) | 登录查找用户 |
| department | UNIQUE(code) | 编码查找 |
| doctor | (dept_id), (user_id) | 按科室/账号查医生 |
| drug | UNIQUE(code) | 药品编码查找 |
| patient | UNIQUE(id_card), (name), (phone) | 挂号台搜索患者 |
| registration | UNIQUE(reg_no), (doctor_id, status), (reg_date) | 工作站待诊队列、当日挂号查询 |
| visit | UNIQUE(registration_id) | 一对一关联保护 |
| prescription | (visit_id) | 查就诊的处方 |
| prescription_item | (prescription_id), (drug_id) | 查处方明细、按药品反查 |

> **新加查询字段时记得加索引**——比如要支持"按手机号搜挂号单"，需要先加 `KEY idx_reg_phone (phone)` 之类的索引；否则全表扫描会慢。

## 7. 初始化数据

`data.sql` 只包含**字典数据**：
- 3 个科室
- 10 个药品
- 5 个示例患者

**用户和医生账号**由 `DataInitializer.java` 在启动时插入（因为要 BCrypt 加密密码，纯 SQL 写不出来）：

| username | password | role | 关联 doctor |
| --- | --- | --- | --- |
| admin | 123456 | ADMIN | （无） |
| reception | 123456 | RECEPTION | （无） |
| doctor | 123456 | DOCTOR | "张医生" / 内科 |

`DataInitializer` 会判断 `sys_user` 表是否为空，**只在首次启动时执行**，避免重启时重复插入。

## 8. 重置数据库的几种姿势

```bash
# 方法 1：删库重建（最干净）
mysql -uroot -proot -e "DROP DATABASE his; CREATE DATABASE his ...;"
mysql -uroot -proot his < backend/src/main/resources/db/schema.sql
mysql -uroot -proot his < backend/src/main/resources/db/data.sql

# 方法 2：仅重跑 schema（保留数据库本身）
mysql -uroot -proot his < backend/src/main/resources/db/schema.sql
mysql -uroot -proot his < backend/src/main/resources/db/data.sql

# 方法 3：用 docker compose（自动跑 schema.sql 和 data.sql）
docker compose down -v       # 删 volume
docker compose up --build    # 起来时自动初始化
```

> 重置后**第一次启动后端**时，DataInitializer 会重新写入 admin/reception/doctor 三个账号。

## 9. 加新表的标准 checklist

每次新加表请逐条核对：

- [ ] 表名小写 + 下划线，与业务名词对齐
- [ ] 主键 `id BIGINT NOT NULL AUTO_INCREMENT`
- [ ] 字符集 `utf8mb4`，排序规则 `utf8mb4_unicode_ci`
- [ ] 必要的 `created_at` / `updated_at`
- [ ] 关联字段命名为 `xxx_id`，加 `KEY idx_*` 索引
- [ ] 状态/类型字段用 VARCHAR + 枚举值
- [ ] 金额用 `DECIMAL(10,2)`
- [ ] 加 `COMMENT '...'` 说明用途
- [ ] 在 `schema.sql` 顶部 `DROP TABLE IF EXISTS` 一致

模板：

```sql
DROP TABLE IF EXISTS xxx;
CREATE TABLE xxx (
    id          BIGINT       NOT NULL AUTO_INCREMENT,
    -- 业务字段
    name        VARCHAR(50)  NOT NULL COMMENT '...',
    -- 状态
    status      VARCHAR(20)  NOT NULL DEFAULT 'XXX',
    -- 时间戳
    created_at  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    PRIMARY KEY (id),
    KEY idx_xxx_name (name)
) ENGINE = InnoDB DEFAULT CHARSET = utf8mb4 COMMENT 'XXX';
```
